# AGIS 系统设计

## 1. 子系统

| 子系统 | 职责 | 主要落地 |
|--------|------|----------|
| UI Shell | SDI 主窗口、分割、Dock、菜单/工具栏/状态栏 | `gis-desktop-win32` |
| Document | 工程模型、保存/加载、脏标记 | 同主程序 |
| Layer | 图层树、类型、可见性、顺序、驱动绑定 | 同主程序 |
| View | 2D 画布与 3D 视口切换、输入路由 | 同主程序 |
| IO | GDAL/OGR、瓦片、点云/模型/流（分阶段） | 库依赖与模块 |
| Georeference | 无 CRS 数据的配准 | 向导与工具 |
| Simulation | 载具状态机与相机 | 3D 视图与更新循环 |

## 2. 依赖关系（自上而下）

Shell → Document / Layer → View → IO  
Georeference 依赖 Layer + View（GCP 拾取）  
Simulation 依赖 View（3D）与可选地形/碰撞数据  

## 3. 与物理目标映射

- **单一可执行目标** `gis-desktop-win32`：聚合上述模块于同一进程；后续可拆出 `agis-core` 静态库，逻辑职责不变，仅物理拆分。

## 4. 关键用例（摘要）

1. **打开工程** → 解析工程文件 → 重建图层树 → 恢复视口与 CRS。  
2. **添加 GDAL 栅格** → 选择文件 → 创建栅格图层 → 加入图层树 → 2D 重绘。  
3. **导入无 CRS 数据** → 启动配准 → 保存变换 → 图层可用。  
4. **切换 3D** → 停用 2D 主循环或并行更新 → 3D 相机与图层可见性。  
5. **启动车辆模拟** → 绑定输入 → 更新 3D 相机与可选物理量。

详细步骤见各 `detailed-design-*.md`。
