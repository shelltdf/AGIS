# 详细设计：导入与地理配准

## 1. 导入路径

- **菜单/拖放**：文件选择器 → 类型嗅探（扩展名 + 魔数）→ 选择驱动 → 创建图层。
- **批量**：目录扫描与多文件队列（路线图）。

## 2. DEM / DOM

- DEM：高程栅格，默认以伪彩或山体阴影显示；可参与 3D 地形。  
- DOM：正射影像，作为栅格图层叠加。

## 3. 矢量（SHP / OSM / GeoJSON）

- OGR 打开；**无 `.prj` 或 CRS 未知** → 提示用户选择 CRS 或进入 **地理配准**。

## 4. 地理配准（无 GIS 信息）

- 用户加载 **控制点**（已知坐标或从参考图层拾取）→ 计算 **仿射/多项式** 变换 → 写入临时 world file 或内存 GeoTransform → 图层显示对齐。

## 5. 点云 / 3D / 倾斜 / 3DGS / 影像序列 / 摄像头

- 各类型 **独立导入向导**：文件或 URL、坐标系、缩放与原点；**视频流** 需 URL 与解码参数（分阶段）。

## 6. 数据转换链路（模型/瓦片/GIS）

- **模型→GIS**：从 OBJ 计算包围盒与中心点，输出 GeoJSON 要素集合，作为 GIS 侧中间层。
- **模型→瓦片**：优先通过“模型→GIS→瓦片”统一链路输出 `xyz/tms/wmts/mbtiles/gpkg/3dtiles`，减少重复实现。
- **瓦片→GIS**：从 `/{z}/{x}/{y}` 目录识别同一最高层级瓦片并拼接成单幅栅格（GeoTIFF），用于后续 GIS 处理与展示。
- **瓦片→模型**：把瓦片索引网格转成平面 mesh（每瓦片一个四边形），并支持按内存预算分批写多个 OBJ，避免超大瓦片集一次性合并失败。
