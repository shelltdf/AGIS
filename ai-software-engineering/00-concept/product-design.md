# AGIS 产品设计（草案）

## 1. 产品定位

**AGIS** 是一款面向专业与进阶用户的 **Windows 本地桌面 GIS 数据编辑与处理软件**，交互与能力对标 **QGIS** 一类通用 GIS 平台：以 **单文档 SDI（Single Document Interface）** 为主壳层——**一个顶层主窗口**承载当前工程与主工作区；新建/打开另一工程时采用 **再开一个顶层窗口** 或 **单窗口内切换当前工程**（实现阶段二选一，须在软件设计文档中固定）。

## 2. 目标用户与成功标准

- **用户**：需要整合栅格、矢量、三维与物联网影像数据，并进行编辑、分析与可视化的用户。
- **成功标准**：在 Windows 上稳定运行；可管理多源图层；2D 画布流畅交互；可按路线图逐步接入 GDAL 等驱动与更多数据类型。

## 3. 壳层类型：SDI

- **SDI**：单一主框架窗口；**左侧**为 **层管理 / 数据源** 停靠区（概念上与「Dock Area」一致：折叠按钮条 + 内容区排版正交，详见 `software-design.md` 与实现映射）。
- **客户区**：默认 **2D 地图画布**；可切换 **3D 渲染视图**（同一主窗口内切换模式，而非默认 MDI 子窗口阵列）。

## 4. 纵向分区（主窗口）

| 区域 | 说明 |
|------|------|
| 菜单栏 | 文件、编辑、视图、图层、导入、分析、模拟、语言、主题、窗口、帮助等（具体以 `02-physical` 与实现为准） |
| 工具栏 | 高频命令；与 Dock 区分，不占 Dock 折叠条 |
| 客户区 | 中央 **2D/3D 视口**；二维主画布交互（中键平移、滚轮以指针为锚点缩放等）在物理规格中细化 |
| 左侧 Dock | **图层树**、数据源添加、符号与可见性 |
| 状态栏 | 坐标、比例、就绪信息；可点击打开 **Log**（与物理规格一致） |

## 5. 数据与图层（能力范围）

### 5.1 图层驱动与来源

- **栅格/影像**：GDAL 可读格式；**DEM / DOM** 导入；**TMS / XYZ** 等瓦片服务；其他 **OGC / 在线服务**（能力分阶段落地）。
- **矢量**：**Shapefile、GeoJSON、OSM** 等；无空间参考或缺失地理信息时，提供 **地理配准 / 标定** 工作流。
- **三维与专项**：**点云**、**3D 模型**、**倾斜摄影**、**高斯泼溅（3DGS）**——以独立图层类型与渲染管线接入，分里程碑实现。
- **序列与视频类**：**无人机连续影像**（时间/空间序列）、**摄像头监控流**——作为媒体或实时图层，分阶段支持。

### 5.2 模拟与漫游

提供 **车辆、飞机、步行、船舶、火箭、卫星/空间站** 等 **第一人称或跟随视角** 的模拟驾驶/飞行模式，与 3D 场景与地形联动；具体物理参数与按键在逻辑/物理文档中定义，分阶段实现。

## 6. 非目标（当前版本声明）

- 不承诺首版即包含 QGIS 全插件生态或同等算法库全覆盖。
- 在线服务与实时流的具体协议集以各里程碑为准，在 `interface-design.md` 中列出对外边界。
